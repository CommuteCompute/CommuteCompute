How the Journey System Works

↻ Data Source Modes

The system operates in two modes to ensure your journey always works:

  • ● Live Data Mode: Real-time transit updates from your state's transit authority API
  • ● Fallback Mode: Static timetable data from GTFS feeds (works without API keys)

The system automatically uses fallback data when API keys aren't configured, ensuring your journey is always accessible.

Location Agnostic Design

This system works across all Australian states and territories:

  • VIC VIC: Trains + Trams (Metro Trains, Yarra Trams)
  • NSW NSW: Trains + Buses (Transport for NSW)
  • QLD QLD: Trains + Buses (TransLink)
  • SA SA: Trams + Buses (Adelaide Metro)
  • WA WA: Trains + Buses (Transperth)
  • TAS TAS: Buses + Ferries (Metro Tasmania)
  • NT NT: Buses (Department of Infrastructure)
  • ACT ACT: Trains + Buses (Transport Canberra)

Transit modes are automatically detected based on your address - no manual configuration needed.

Journey Calculation Process

The system builds your journey through these steps:

  • 1. Geocode home and work addresses (free Nominatim service)
  • 2. Detect state and select appropriate transit authority
  • 3. Find nearby transit stops using GTFS data
  • 4. Calculate walking times between stops
  • 5. Build multi-modal journey with connections
  • 6. Apply real-time updates if API keys configured

Transit Mode Icons

Visual indicators for each journey segment:

  • Train - Heavy rail services
  • Tram/Light Rail - Street-level rail
  • Bus - Bus services
  • [Ferry] Ferry - Water transport
  • Walking - Walking segments

Enabling Live Data (Optional)

To switch from fallback to live data:

  • 1. Go to Admin Panel → API Settings tab
  • 2. Register for your state's transit API (links provided)
  • 3. Enter your API credentials
  • 4. System automatically switches to live data

Remember: API keys are optional! The system works great with fallback data.
